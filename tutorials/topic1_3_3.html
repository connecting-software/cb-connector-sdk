    Implementing DataHandlerFactory.cs

    Creating the new class

    If you have choosen to create your connector solution using Visual Studio Project Template, you can skip this step.

    Start by creating a new HandlerFactory class. You need to implement methods from the the CB.Connector library, so a reference to this library is required in your class.

    using CB.Connector.Classes;
using CB.Connector.Command.DataHandler;

    Implement the TryCreate method

    First implement the TryCreate method. This method is called by CB Server to check if your custom connector supports a specific SQL data manipulation command, and if so, to get a new data handler object, as shown in the following example:

    public override bool TryCreate(out Where handler)
{
    //Where clause is supported by the custom connector.
    handler = new WhereHandler(Session);
    return true;
} 

public virtual bool TryCreate(out Having handler)
{
    //Having clause is not supported by the custom connector.
    handler = null;
    return false;
}
